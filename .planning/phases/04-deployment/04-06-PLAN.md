  <action>
Perform clean deployment test and document results:

1. **Prepare clean environment:**
```bash
# Stop any running containers
docker-compose down -v

# Remove all images (optional - for full clean test)
docker-compose down --rmi all -v

# Verify clean state
docker ps -a | grep rag || echo "No containers"
docker images | grep rag || echo "No images"
```

2. **Run setup script:**
```bash
# Time the setup
start_time=$(date +%s)
./scripts/setup.sh
end_time=$(date +%s)
echo "Setup time: $((end_time - start_time)) seconds"
```

3. **Start services:**
```bash
docker-compose up -d

# Wait for health checks
sleep 30

# Verify all services are healthy
docker-compose ps
```

4. **Test endpoints:**
```bash
# Test ChromaDB
curl -f http://localhost:8001/api/v1/heartbeat
echo "✓ ChromaDB healthy"

# Test Backend
curl -f http://localhost:8000/api/health
echo "✓ Backend healthy"

# Test Frontend
curl -f http://localhost:80/ > /dev/null
echo "✓ Frontend healthy"

# Test API docs
curl -f http://localhost:8000/docs > /dev/null
echo "✓ API docs accessible"
```

5. **Create test results document:**
Create `DEPLOYMENT-TEST.md` with:
- Test date and environment
- Setup time (target: <10 minutes)
- Service status
- Health check results
- Any issues encountered
- Resolution steps

Document all findings and ensure deployment is successful.
