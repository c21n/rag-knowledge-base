  <name>Create backend Dockerfile</name>

  <action>
Create `backend/Dockerfile` with the following specifications:

1. **Base image**: Use `python:3.11-slim` for production optimization

2. **Working directory**: Set `/app`

3. **System dependencies**: Install any needed system packages (e.g., gcc for compiled dependencies)

4. **Python dependencies**: 
   - Copy requirements.txt first (for layer caching)
   - Run `pip install --no-cache-dir -r requirements.txt`

5. **Application code**:
   - Copy entire src/ directory
   - Copy alembic/ directory (for migrations)
   - Copy alembic.ini

6. **Environment**:
   - Set PYTHONUNBUFFERED=1
   - Expose port 8000

7. **Entrypoint**: Use uvicorn with production settings:
   - `uvicorn src.main:app --host 0.0.0.0 --port 8000`
   - Include `--workers 4` for production

8. **Optimization**:
   - Use .dockerignore to exclude unnecessary files (venv, __pycache__, .git, etc.)
   - Keep image size under 1GB

Also create `backend/.dockerignore`:
```
.venv/
__pycache__/
*.pyc
*.pyo
*.pyd
.Python
.git/
.gitignore
.pytest_cache/
.coverage
htmlcov/
.env
*.log
data/
uploads/
```

<task type="auto">
  <name>Test backend container</name>
  <files>backend/Dockerfile</files>
  <action>
Test the backend Docker container:

1. Build the image: `docker build -t rag-backend:test .`

2. Run the container with test environment:
```bash
docker run -d \
  --name rag-backend-test \
  -p 8000:8000 \
  -e OPENAI_API_KEY=test-key \
  -e DATABASE_URL=sqlite:///./data/app.db \
  rag-backend:test
```

3. Wait 5 seconds for startup

4. Test health endpoint:
```bash
curl -f http://localhost:8000/api/health || echo "Health check failed"
```

5. Check container logs:
```bash
docker logs rag-backend-test | tail -20
```

6. Clean up:
```bash
docker stop rag-backend-test && docker rm rag-backend-test
```

7. Report image size:
```bash
docker images rag-backend:test --format "{{.Size}}"
```

Document any issues found and ensure the container starts successfully.
  </action>
